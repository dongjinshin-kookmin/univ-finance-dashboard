# 기본설정 페이지 + 전역 기준군(벤치마크) 체계 설계

교비회계 수지분석 대시보드(`dashboard/src/app.js` · 344교 · 2016~2024)에 **기본설정 페이지**와
**전역 기준군(전국 / 수도권 / 경쟁대학)** 체계를 추가하기 위한 설계 문서. 코드는 포함하지 않으며,
모든 결정은 실측 근거(파일·라인)와 함께 기술한다.

> 실측 도구·환경: `build/dashboard_data.json`(생성 2026-07-20), `dashboard/src/app.js`(2958줄),
> `dashboard/src/styles.css`, `dashboard/src/index.html`, `dashboard/build_html.py`, `etl/config.py`.

## 0. 핵심 결정 요약

| 항목 | 결정 |
|---|---|
| 지속성 | `localStorage` 단일 키 `kmu-dash:settings`(스키마버전 payload 내장) + **전 경로 try/catch → 인메모리 폴백** + **JSON 내보내기/가져오기**(file:// Safari 대비). 학교는 **이름(name)으로 저장**해 ETL 재빌드에도 안정. |
| ETL 영향 | **없음.** 모든 기준군 집계는 인메모리 병렬배열에서 실시간 계산(기존 `cohortStats` 선례). `agg` 사전집계는 typeFilter와 불일치하므로 **계속 미사용**. |
| 기준군 집계 | **비율지표=중앙값(p50) 주선 + p25~p75 밴드**, **금액지표=구성비/1인당 백분위**(원값 평균 비교 금지). 평균은 보조. **경쟁대학(N≤7)은 밴드 대신 개별선 + 그룹 중앙값 기준선**. |
| typeFilter × 기준군 | 전국·수도권=**typeFilter 적용**, 경쟁대학=**적용 안 함(명시 선택 우선)**, 메인대학=**항상 개별 표시**(집계군 편입은 필터 준수). |
| 색 계약 | `--kmu`(#004F9F)=제품 크롬 + 국민대 엔티티색 **영구 고정**. **강조(굵기·★·라벨)와 색을 분리** → 메인대학은 강조로 표시, 국민대는 어디서든 파랑 유지. 신규 `--main` 토큰은 히어로/워크스페이스 등 "분석주체" 표면에만. |
| 탭 반영 | 홈=설정요약+버튼 / 개요=기준군 백분위 토글 / 구조·계정=3중 벤치마크를 전국·수도권·경쟁으로 교체 / 시계열=3선 오버레이+전국밴드 / 비교=메인+경쟁 프리시드 / 위기·전망·시뮬=기준군 요약행. |
| WBS | **P0(핵심·직렬)** → **P1 설정·홈 ∥ P2α 상단탭 ∥ P2β 하단탭**(각 worktree 격리) → **P3 검증**. 공유코드는 전부 P0에 선적재해 병렬 충돌 제거. |

## 1. 실측 현황 (설계 근거)

### 1.1 데이터 계층 (`dashboard_data.json`)
- 최상위 키: `meta, schools(344), accounts, rows(2088), v{in,ex}, kpi(10), agg{all,c100}, validation, ext, sim`.
- `schools[i]` = `{n, c100, kmu, sido, region, type, scale, scaleProxy, enroll{year:val}}`.
  - `type` 분포: **대학 157 / 전문대학 126 / 대학원대학 41 / 사이버대학 20**.
  - `region` 값: **수도권 153 / 지방권 131 / 광역권 60** (※ "전국"이라는 값은 없음 → 전국 = 전체).
  - `type=='대학'` 중 region: **수도권 64 / 지방권 66 / 광역권 27**.
- 국민대학교: `id=52`, `kmu=true`, `c100=true`, `서울·수도권·대학·대규모`, 재학생 ~14,486(2024).

- **`agg{all,c100}`는 KPI별 `{year:[p25,p50,p75,mean]}` 사전집계이나 `app.js`·`charts.js`·`sim_model.js` 어디에서도 참조 없음 → 완전 미사용(dead).**
  - 근거: `grep -rn "\.agg" dashboard/src/*.js` → 매치 없음.

## 2. 상태·지속성 설계

### 2.1 설정 스키마
신규 서브객체 `S.settings`(payload에 스키마버전 내장):

```
S.settings = {
  v: 1,                              // 스키마 버전(마이그레이션 키)
  mainSchoolName: "국민대학교",       // ★ id가 아닌 이름으로 저장(재빌드 안정)
  competitorNames: [                 // 1~7개, 가나다 정렬 저장
    "경희대학교","세종대학교","숙명여자대학교","숭실대학교","중앙대학교","한양대학교"
  ],
  typeFilter: ["대학"]               // 전역 집계 대상 형태(기본 대학만)
}
```

파생 런타임 전역(설정 적용 시 재계산):
`MAIN_ID`(=이름→id 해석), `COMP_IDS[]`, `TYPE_SET`(Set), 색맵(`schoolColorMap`), `nonMain[]`.

**결정 근거 — 이름 저장:** `sid`는 `schools[]`의 위치 인덱스(ETL가 학교 추가·정렬 변경 시 이동). 이름은 학교마스터의 고유·불변 식별자(`build_school_master.py`). 재빌드 후에도 안정. 트레이드오프: 학교 개명 시 "제거"로 처리 → 로드 시 경고 후 폴백(§7-5).

### 2.2 지속성: localStorage + 폴백
- **단일 키** `kmu-dash:settings`. 파일명(dashboard.html / dashboard_lite.html)에 의존하지 않는 **고정 앱ID**를 키에 사용 — Chrome이 모든 file:// 를 `null` 오리진으로 공유하는 특성에서, 두 번들이 **의도적으로 같은 설정**을 공유하도록(개념적 설정이므로 무해). 버전은 payload 내부.
- **전 경로 try/catch:** `window.localStorage` **접근 자체가 예외를 던질 수 있음**(Safari file://, 프라이빗 모드) → feature-detect 프로브부터 감싼다. 읽기 실패·파싱 실패·quota 초과 모두 → **인메모리 세션 설정**으로 폴백하고 조용한 배너 표시("이 브라우저에서는 설정이 저장되지 않습니다 — 내보내기로 백업하세요").
- **JSON 내보내기/가져오기:** 설정을 텍스트로 복사/붙여넣기(클립보드). file:// Safari처럼 localStorage가 막힌 환경의 **범용 지속 우회로**이자 기기 간 공유 수단.

> **file:// localStorage 브라우저 거동 (⚠ 일부 추정 — P3 실측 권장):**
> - Chrome/Chromium: 동작하나 **모든 file:// 페이지가 `null` 오리진 단일 저장소 공유** → 번들 간 충돌 가능(고정키 공유로 흡수).
> - Firefox: 최근 버전은 파일 경로별 파티션 — 대체로 동작.
> - Safari: file://(및 프라이빗)에서 접근 시 **SecurityError 던질 가능성 높음**.
> 안전 래퍼(try/catch + 인메모리 폴백 + 내보내기)가 **모든 경우에 설계를 정상화**하므로 브라우저별 확정 없이도 정합.

**기각한 대안:**
- URL 해시/쿼리 인코딩 — 사용자가 북마크하지 않으면 휘발. 단, **공유용 딥링크의 선택적 2차 수단**으로는 가치 있음(향후).
- IndexedDB — 동일 file:// 오리진 문제 + 비동기 복잡. 이점 없음. 기각.

### 2.3 무효화·재렌더 전략
- 설정 변경 → 신규 `applySettings(next)`: ①스키마 검증·정규화(§7 엣지) ②파생 전역 재계산(`MAIN_ID`, `COMP_IDS`, `TYPE_SET`, 색맵) ③`S.types`를 `typeFilter`로 동기화 ④지속(try/catch) ⑤`refresh()`.
- **무캐시 단일 재렌더**(§1.2)이므로 탭별 무효화 불필요. 모든 탭이 `MAIN_ID`·기준군 함수를 **실시간 참조**하면 재렌더 1회로 전파.
- 주의: `KMU_ID`(`var`, 27행) → **가변 `MAIN_ID`(`let`)** 로 승격. `filteredSchoolIds`의 강제 포함(236행)을 `MAIN_ID`로 일반화. 색맵·`nonMain`은 메인 변경 시 재계산(§5).

### 2.4 순수 클라이언트 처리 범위
- 전국(≤157)·수도권(≤64)·경쟁(≤7) 집계는 전부 인메모리 실시간(이미 `cohortStats`가 필터모집단으로 수행). **ETL·JSON 변경 0.**
- **`agg` 사전집계는 사용 금지 유지:** `agg`는 전 타입(all/c100) 고정 통계라 **typeFilter(대학만)와 불일치** → 대학만 전국 밴드에 못 씀. 실시간 계산이 정답. (선택적 정리: 미사용 `agg` 제거 시 번들 소폭 축소 — 필수 아님.)

## 3. 기준군 의미론

### 3.1 세 기준군 모집단 정의 (신규 엔진 `benchmark.js`)
`benchmarkPops()` → 아래 id 집합 반환(설정·typeFilter 구동, **세션 사이드바 칩과 독립**):

| 기준군 | 모집단 | typeFilter | 메인대학 처리 |
|---|---|---|---|
| 전국 | 전체 학교 ∩ typeFilter | **적용** | 집계 편입은 필터 준수, 표시는 항상 개별 |
| 수도권 | 전국 ∩ `region=='수도권'` | **적용** | 동일 |
| 경쟁 | `COMP_IDS`(1~7 명시) | **미적용** | 메인은 후보에서 제외(§7-1) |

> **결정 — 전역 기준군은 사이드바 필터와 별개 축.** 전국은 정의상 전 권역을 포함하므로 `S.regions/S.scales`를 무시하고 `typeFilter`만 적용. 사이드바 칩(권역·규모·코호트)은 **탐색용 "필터 모집단"**(산점도·랭킹·분포의 우주)으로 존속. `typeFilter`의 **기본값만** 설정에서 `S.types`로 주입(기본 대학만). 두 모집단을 문서·UI에서 명확히 구분해 혼동 방지.

### 3.2 집계 방식 — 지표 성격별 (재무 규모 편차 대응)
344교는 대규모↔소규모 편차가 극심(우편향). **원값 평균은 거대교에 지배됨.**

- **비율지표(pct KPI: 등록금의존율·운영수지율·법인전입금비율·장학금지원율·인건비부담률·이월금비율·교육비환원율):**
  **중앙값(p50) 주선 + p25~p75 밴드**를 표준. 평균은 보조(툴팁·상세). 근거: 편향에 강건, 사학진흥재단 관행(분위 보고), 기존 시계열 밴드와 일치.
- **금액지표(krw: 운영수지·적립금순증·계정 금액):**
  단일교 원값을 코호트 평균/중앙값과 비교하면 규모효과로 오도. **원값 평균 비교 금지.** 대신 **구성비(share-of-total) 백분위** 또는 **1인당(per-student)** 로 벤치마크(기존 `benchmarkGroups`·`perStudentCard`가 이미 이 방식). 중앙값 기반 백분위.
- **경쟁대학(N≤7) 특례:** 6점의 p25/p75는 노이즈. **밴드 대신 각 경쟁교 개별선·개별점(이름표) + 그룹 중앙값 1선**만. 대규모 전국·수도권만 밴드 유효.

### 3.3 기존 자산과의 관계 (대체 vs 병존)
- **`cohortStats`/`stats` 재사용:** 모집단만 파라미터화하면 그대로 사용(중앙값·평균 동시 산출). 신규 엔진은 얇은 래퍼.
- **`benchmarkGroups`(1014행) = 교체:** 그룹 정의 `[모집단, 같은규모, 같은권역]` → **`[전국, 수도권, 경쟁]`** 로 변경(계정·구조 탭 3중 벤치마크가 전 앱과 일관되게). 단, **`perStudentCard`의 "같은 규모 백분위"(1074행)는 존치** — 규모정규화 비교로 여전히 유효.
- **`agg` = 미사용 유지**(§2.4).
- **`S.cohort`(전체/100개) = 병존:** 탐색 필터 축으로 남김. 전역 기준군과 직교.

## 5. 메인 대학 변경 정책 (색·브랜드 계약)

**계약: 파랑 #004F9F = 국민대 전용.** 다른 학교를 이 파랑으로 칠하면 계약 위반·오도.

**핵심 원리 — 색(엔티티 정체성)과 강조(분석 주체)를 분리.**
현재는 `emphasize: sid===KMU_ID` **와** 색=파랑이 결합(1454, 1553행 등). 이를 분리:
- `emphasize`(굵은 선 · ★ · 라벨 · 마커) → **`MAIN_ID` 추종**.
- 색 → `schoolColor(sid)`: **`국민대ID → var(--kmu)` 항상**(어디서든 파랑 유지, 경쟁으로 등장해도), 그 외 `series-2..8` 순환. 메인이 국민대가 아니면 **자기 시리즈색 + 강조**.
  - 예: 메인=중앙대, 국민대=경쟁 → 중앙대(자기색·굵게·★), 국민대(파랑·보통). **파랑은 국민대에만, 강조는 메인에.** 계약·의미 동시 충족.

**제품 크롬 vs 분석 주체 분리:**
- 제품 크롬(내비 활성·포커스링·버튼·아이브로우, `--ring:var(--kmu)`)은 **KMU Blue 영구 고정** — 국민대가 만든 도구의 저작 브랜드(계약 위반 아님).
- 신규 **`--main` 토큰**: 메인=국민대 → `var(--kmu)`, 메인≠국민대 → **비(非)파랑 강조색**(딥슬레이트/잉크 계열 권장 — 경쟁교 시리즈색과도 구분). 사용처는 **분석주체 표면에 한정**: 히어로 그라데이션·워크스페이스 아바타·홈 요약카드 액센트. **파랑 제품 내비는 그대로.**

**국민대 특화 요소:**
- 히어로 80주년 캠퍼스 사진: `data-main` 바디 속성으로 게이트. `data-main="kmu"` → 80주년 사진+문구, 그 외 → **중립 그라데이션 히어로**(메인대학명·KMU 사진/80주년 문구 제거).
- 사이드바 `KMU`·워크스페이스·크럼 "국민대학교": **분석 주체(메인대학명)로 일반화**. 단 푸터에 "KMU 재정분석" 도구 브랜드 소자 존치(저작=국민대 유지).
- 하드코딩 문자열 "국민대" 다수(510, 551, 617, 630행 등): 메인대학명 변수로 치환(P0에서 `MAIN_NAME` 도입).

## 7. 엣지 케이스

1. **경쟁대학에 메인 포함 시도:** 경쟁 후보에서 `MAIN_ID` 제외. 메인을 경쟁 포함 학교로 변경 시 **경쟁에서 자동 제거 + 경고**("메인 대학은 경쟁에서 제외"). 두 집합 항상 서로소.
2. **폐교/데이터 부재 학교 선택:** 허용하되 표시. `latestOf`(81행)로 최신 가용연도 폴백, `stats()`가 null 제외 → 결측 연도는 선이 끊길 뿐 무크래시. 설정에서 "최근 데이터 없음(최신 20xx)" 배지.
3. **typeFilter로 메인 제외(예: 메인=사이버대, 필터=대학만):** 메인은 **자기 선/값 항상 개별 표시**(강제 포함 일반화), 단 **전국·수도권 집계 모집단에선 제외**(필터 준수·자기오염 방지). 영향 탭에 1줄 주석("메인 대학은 현재 집계 필터에서 제외되어 개별 표시"). 자동으로 필터에 메인 타입 추가는 **기각**(명시 필터 존중). 교차타입 백분위엔 캐비엇.
4. **localStorage 스키마 버전:** payload `{v:1,…}`. 로드 시 `v` 없음/과거 → 마이그레이션 맵(불가 시 폐기+기본+알림). 파싱 실패 → try/catch → 기본. 미래 상위 `v` → 알려진 필드만 읽고 저장 전까지 다운그레이드 쓰기 금지.
5. **경쟁 이름 해석 실패(개명/삭제):** 로드 시 이름→id 해석, 미해석은 드롭+토스트("일부 경쟁대학 제외: XXX"). 해석 0개 → 기본 6교 복원. 메인 미해석 → 국민대(없으면 schools[0]) 폴백.
6. **수도권 집계 공집합 등:** `stats()` null 반환 → "데이터 없음" 우아 처리(기존 경로).

## 8. WBS (작업 분해 · 병렬 구조 · 담당 티어)

**단일 파일(app.js 2958줄) 병렬 편집 = 충돌 위험.** 해소 원칙: **① 공유·전역 코드를 전부 P0에 선적재**, ② 이후 단위는 **서로소 렌더 함수/신규 파일**만 편집, ③ 병렬 app.js 편집기는 **worktree 격리 후 머지**.

```
P0(핵심·직렬) ──▶ P1 설정·홈  ┐
              ├─▶ P2α 상단탭   ├─(worktree 격리·병렬)─▶ P3 검증(직렬)
              └─▶ P2β 하단탭  ┘
```

| 단계 | 작업 | 만지는 파일 | 병렬 | 티어 |
|---|---|---|---|---|
| **P0** | `MAIN_ID`(가변)·`MAIN_NAME` 일반화, `filteredSchoolIds` 강제포함 일반화, `S.settings`+지속성 모듈, 기준군 엔진, `schoolColor`/emphasis 분리·`--main` 토큰, 공유 컴포넌트(`BenchmarkBar`·`benchCard` 교체·`schoolPicker`), TABS에 `settings` 항목+디스패치 | `app.js`(20~250, 391~427, 공유헬퍼), **신규 `settings.js`·`benchmark.js`**, `styles.css`(토큰), `index.html`(script·톱니 슬롯) | 직렬(선행) | **EX(opus-executor)** |
| **P1** | `renderSettings` 신규, 홈 요약카드, `HOME_CARDS`/히어로 조건부 | `app.js`(`renderHome` 446~501 + 말미 `renderSettings` 추가), `styles.css`(**말미 append**) | P2와 병렬 | **EX** |
| **P2α** | 상단탭 기준군 통합 | `app.js`(`renderOverview` 506 / `renderStructure` 735 / `renderAccounts` 1178 / `renderTimeseries` 1397 본문) | ∥ P1·P2β | **EX** |
| **P2β** | 하단탭 기준군 통합 | `app.js`(`renderCompare` 1478 / `renderCrisis` 1687 / `renderOutlook` 1849 / `renderSimulation` 1992 본문) | ∥ P1·P2α | **EX** |
| **P3** | `build_html.py` 번들 재생성, file:// 스모크(Chrome/Safari/FF), 지속성 폴백·엣지·색계약(비국민대 파랑 금지) 감사, 기준군 의미 사인오프 | 소스 편집 없음(수정 시 국소), 빌드/스모크 | 직렬(수렴) | **EX + runner(빌드·조회) + deep-reasoner(의미 검토)** |

**의존·병렬 규칙:**
- P1·P2α·P2β는 P0 완료 후 각각 **자기 렌더 함수 본문에 호출만 추가**(공유 헬퍼는 P0 소유) → 라인영역 서로소. 그럼에도 같은 파일이므로 **각자 git worktree(`isolation:"worktree"`)에서 작업 후 머지** 권장.
- `styles.css` 충돌 회피: **토큰·변수는 P0**, 컴포넌트 스타일은 P1이 **파일 말미 append-only**.
- `build_html.py`·JSON·ETL: **변경 없음**(순수 클라이언트). 데이터 계약 불변.

## 9. 리스크 · 미확인 사항

- **(추정) file:// localStorage 브라우저별 거동**(특히 Safari SecurityError) — P3 실측 권장. 단, try/catch+인메모리+내보내기 래퍼가 결과를 정상화하므로 설계 정합성엔 영향 없음.
- **(확인) `agg` 미사용** — 재확인 완료(`grep` 무매치). 배선 금지 명문화.
- **경쟁대학 소규모 통계 불안정** — 밴드 대신 개별선으로 회피(§3.2).
- **폐교/결측 12교**(운영수지율 2024 기준 332/344 보유) — null-skip으로 흡수.
- **색계약 회귀 위험** — 병렬 편집 중 비국민대에 `var(--kmu)` 오배정 가능 → P3에서 전용 감사(강조≠색 분리 준수 확인).
- **단일 파일 병렬 편집 머지 리스크** — worktree 격리로 완화하되, 여의치 않으면 P1·P2 **직렬화**가 안전한 폴백.
