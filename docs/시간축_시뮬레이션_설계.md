# 시간축 시뮬레이션 — "20년 관찰" 확장 설계 (deep-reasoner, 2026-07-22)

감축 시나리오 파라미터(r·β·π·γ·λ·법인지원) 변경 결과를, **2026년부터 2045년까지 20년을 천천히 관찰**하며
위험/안정 대학이 페이지 전체에서 쉽게 구분되도록 하는 시간축 확장. 기존 정착판정 레이어
(`docs/폐교위험_판정_설계.md`, `build/d14_report.md`)를 **무수정 유지**한 채 위에 얹는다.

핵심 원리: **정착판정(closureGrade)은 "이 시나리오면 결국 어떤 등급인가"라는 1회 전망**이고,
시간축은 그 전망에 이르는 **경로 grade(t)** 를 분해해 보여준다. 둘은 같은 검증된 매트릭스를 공유하므로
모순 없이 공존한다.

## 1. grade(school, year) 의미론

### 1.1 정의 — 검증된 매트릭스의 연도 스냅샷 재적용

각 연도 t의 등급은 **§2 게이트 매트릭스·오버라이드를 연도 t의 스냅샷에 그대로 재적용**한다. 새 임계값·새 로직 없음.

```
grade(t) = applyOverrides(
             matrixGrade( marginClass(M(t)), bufferClass(R(t), reserveMonths(t)) ),
             M(t), fill(t), reserveMonths(t) )
```

| 입력 | 정의 (연도 t) | 산출 |
|---|---|---|
| **M(t)** 운영수지율 | `recalcSeries[t].primed['운영수지율']` — 그 해 단년 primed 수지율 | r=0이면 전 연도 = 관측 t0값(평탄) |
| **reserve(t)** 완충잔액 | rollforward 경로: `reserve(2025)=reserve_avail`, `reserve(t+1)=reserve(t)+primed운영수지(t)+corp_addl` | 연도별 잔액 배열 |
| **R(t)** 잔여 runway | `depletionYear−t` (소진연도 존재 시), 없으면 `∞`; `reserve(t)≤0`이면 `R(t)=0` | **관측시점 t에서 본 잔여 여력**(t 진행 → 감소) |
| **reserveMonths(t)** | `reserve_month0 × reserve(t)/reserve(2025)` — t0값을 정착앵커로 고정, 소진분수로 스케일 | 유동성 오버라이드 ④⑤ 구동 |
| **fill(t)** 충원율 | λ-off: `f0`; λ-on: `f0·(1+λ·cumDrift(2025→t))` — **그 해까지의** 드리프트(정착값 아님) | 오버라이드 ①② 구동 |

**정착판정과의 완충축 차이(의도된 것):** 기존 `closureGrade`의 R은 t0에서 잰 **총** 여력(오늘 재고 적정성 1회 평가).
grade(t)의 R(t)는 **연도 t에서 본 잔여** 여력이라 소진연도가 다가올수록 감소 → 등급이 서서히 악화한다.
이 시간적 악화가 곧 사용자가 요구한 "천천히 변화 관찰"의 실체다.

### 1.2 검증 매트릭스·오버라이드와의 일관성 (t0 판정 일치)

- **회귀 앵커(r=0, t=2025):** M(2025)=관측 운영수지율, R(2025)=`depletionYear−2025`(= 기존 closureGrade의 t0 runway와 동일식),
  reserveMonths=`reserve_month0`, fill=`f0`. → **D6가 검증한 관측모드(projection=null, 단년 스냅샷) 입력과 정확히 동일** →
  `grade(2025)|r=0` 는 검증된 판정과 일치한다. (관측모드가 이미 "단년 스냅샷에 매트릭스 적용"이므로 grade(t)는
  정착판정보다 **오히려 검증대상에 더 가깝다** — D6는 폐교교 T-1/T-2 단년 스냅샷으로 4/4 민감도를 확보했다.)
- **정착 수렴:** M(t)는 코호트 전개(≈6년) 후 평탄화 → `mean(M(2033..2035)) ≈ closureGrade.M(horizon=10)`.
  정착구간 집계 등급수 ≈ 기존 `closureAggregate`(§5 검증 T3로 확인). 완충축만 설계상 갈라진다(위 1.1).
- **결론:** `closureGrade`/`closureAggregate`는 **손대지 않는다**. 시간축은 별도 함수로 추가하고, 회귀는 테스트로 고정.

### 1.3 등급 진동(flapping) — 히스테리시스 불요 (증거 기반)

모델은 **결정론적**이고 grade(t) 입력은 **거의 단조**다:
- 감축 전파: immediate=상수 r → ΔH 단조하락 후 평탄; linear ramp → 단조. **회복 반등 없음.**
- runway R(t)=depletionYear−t 는 매년 −1 **엄격 단조 감소**; reserveMonths 단조 감소; fill 드리프트 단조.
- 유일한 비단조: π↑(등록금)과 인원↓가 경합해 M(t)가 U자/혹형이 되는 경우 → 임계 교차가 두 번 일어나
  "하락 후 회복"이 **실제로 발생**한다. 이는 노이즈가 아니라 **관찰해야 할 진짜 전이**이므로 억제하지 않는다.

→ 확률적 지터가 없으므로 **히스테리시스 기본 미적용**. 단, D-검증(§5 T4)에서 *1년 반짝 뒤 즉시 원복*하는
반전이 344교×시나리오 스캔에서 관측되면 선택적 가드를 켠다:

> **확정지연 규칙(옵션, 기본 OFF):** 등급 변경은 **연속 2년 지속**될 때만 커밋; 1년만 유지 후 되돌면 직전 등급 유지.
> 상태 1개(직전 pending)만 들고 O(1). 근거 없이 켜지 않는다(불필요한 복잡도 회피).

## 2. 20년 지평(2026~2045)의 정직성

백테스트는 4년 검증인데 20년을 보여달라는 요구 → **어디까지 검증이고 어디부터 기계적 외삽인지**를 축 위에 명시한다.

### 2.1 3구간 + 정직성 경계선 (달력연도 고정)

| 구간 | 연도 | 근거 | 렌더 |
|---|---|---|---|
| **검증(Validated)** | 2026~2029 | 백테스트 4년 예측검증 범위 내 | 실선·불투명·풀 신뢰 |
| **투영(Projected)** | 2030~2040 | 백테스트 밖이나 인구추계 데이터 실재(2040까지) | 실선 + 옅은 해치 + "투영" |
| **관찰용 외삽(Extrapolated)** | 2041~2045 | 인구데이터 종료·price_cap 캐리포워드·경험검증 전무 | 강한 해치 + "관찰용 외삽" 라벨 |

경계선은 **달력 고정**(2029.5·2040.5 세로 구분선) — t0/rStart를 움직여도 위치 불변. 모든 시간축 뷰(히어로·incidence)에 동일 배경.

### 2.2 불확실성 밴드 — 시간에 따라 확대 (상수 σ 금지)

기존 `residualBand`는 상수 `mape_rev`를 쓴다 → 20년에 그대로 쓰면 **오차 누적을 과소표현**. 시간확대 적용:

```
σ(t) = mape_rev · sqrt( max(1, (t − 2025) / h_bt) ),   h_bt = 4 (백테스트 지평)
```

t≤2029 → 계수 1(검증구간). t=2045 → √(20/4)=√5≈2.24배. **통계적 CI가 아니라 "관찰 보조 불확실성 표시"로 라벨**한다
(N=4~6 겸손 원칙 승계 — AUC/CI 과신 금지). bt=null 학교는 개별밴드 생략(기존 `closureBoundary`와 동일)하고
집계밴드(atRiskRange)만 bt 보유교의 M을 ±z·σ(t) 섭동해 산출 → 히어로의 위험경계선이 시간에 따라 **벌어지는** 것으로 보인다.

### 2.3 드라이버 꼬리 처리 (데이터 종료 이후)

- **price_cap 2027+:** `piRateOf` 이미 캐리포워드(마지막 공고 0.0319 유지) — 코드 확인. π=freeze(기본)면 무영향.
  π≠freeze 구간의 등록금 인상분은 **외삽구간 스타일로 음영 + "상한 캐리포워드 가정" 라벨.**
- **인구드리프트 2041+:** `cumDrift`가 미존재 연도 `d=0`으로 처리 → **현재 코드는 자동 "마지막값 유지(동결)"**.
  대안: (a) 동결[현재], (b) 마지막 YoY율 지속, (c) 감쇠. **기본 = (a) 동결 + 강한 라벨** 채택 —
  근거: 데이터 없는 감소를 주입하면 근거 없이 위험을 과장한다. "2041+ 인구추계 데이터 없음(마지막값 유지)"를
  fill 그래프·타일맵 툴팁에 명시. (b)는 `driftTail:'lastRate'` **감쇠 토글(민감도용)** 로만 노출(기본 OFF).
  → fill(t)가 2040에서 꺾여 평탄해지는 "킹크"는 **의도된 정직성 신호**이며 라벨로 이유를 명시.
- **s_k 20년 유효성:** 12-tap 잔존은 코호트별 고정 프로파일이라 horizon과 무관하게 재적용 — "만료" 없음.
  가정은 **잔존 구조의 20년 정상성**(학과 통폐합 등 구조단절 미모형). 낮은 증분 리스크로 §7에 명시.

### 2.4 관찰창·기저연도 결정 — rStart 디커플 (가법·후방호환)

관찰창은 **달력 2025(앵커)→2045 고정**, 슬라이더는 2026~2045. 문제: 현재 `reductionOf`·`project`는 t0가
"기저연도 + 감축시작 + π/드리프트 원점"을 겸해, t0를 2030으로 옮기면 2025~2029가 사라진다.

**결정(1개소 가법 변경):** `reductionOf`의 온셋 원점을 `params.rStart ?? params.t0`로 한다(게이트·램프 원점 모두).
- 시간축 파라미터 = `{ …userP, t0:2025, horizon:20, rStart:userP.t0 }` — 즉 기존 t0 슬라이더값을 **감축 온셋(rStart)** 으로 재해석하고
  기저/π/드리프트 원점은 2025에 고정. 관찰창은 항상 2025~2045.
- **후방호환:** `rStart` 미지정 시 `=t0` → 기존 전 호출 **완전 동일**. 개별대학 팬차트(t0 겸용)는 무변경.
- π·드리프트 원점은 t0(=2025 base)에 남아 달력 정합 유지. 온셋만 rStart로 분리.

### 2.5 수치 정합 — "앞 10년 동일" 증명

`project`의 연도별 산식(컨볼루션·piFactor·base경로)은 **어느 것도 horizon에 의존하지 않는다**; horizon은 루프 상한만 정한다
(`sim_model.js:139-184`). ∴ `project(horizon=20).rows[i] === project(horizon=10).rows[i]` (i: 2025..2035) — 비트동일.
`recalcSeries`·grade(t)도 행 단위라 앞 구간 동일. **회귀테스트 T2로 고정**(deep-equal). rStart==t0==2025 기본에서는
전 파이프라인이 오늘과 동일.

## 3. "천천히 변화 관찰" UX — 채택 시각화 조합

사용자 요구 이중조건 = **구분하기 쉬움(위험 vs 안정)** + **천천히 관찰(20년 변화)**. 후보 (a)~(d)를
"얼마나(stock)·누가(identity)·언제(timing)·어디(region)"로 매핑해 계층 구성한다.

### 3.1 채택 조합과 근거

| 계층 | 뷰 | 답하는 질문 | 채택 이유 |
|---|---|---|---|
| **T1 히어로** | **(a) 등급 스택 영역차트** 2026~2045 | 얼마나(전국 stock 추이) | 좌→우 스캔 시 녹색 축소·주황/적색 팽창 = **20년 완만변화의 시각적 실체**. 시간 백본. |
| **T2 스냅샷** | **(b) 슬라이더 연동 권역 타일맵**(1교=1타일) | 누가·어디 | 개별 식별 + 권역 그룹핑으로 (d) 흡수. 슬라이더 주 타깃. 등급변화 타일 **하이라이트**. |
| **T3 발생** | **(c) 위험진입 연도분포**(연도별 newAtRisk 막대) | 언제(파고 타이밍) | 스택(stock)과 상보. "2038년 N개교가 선을 넘는다"를 명시. 히어로 x축에 얇은 막대로 겹침. |
| (d) 권역 | 타일맵 그룹핑 + 선택적 소계 토글 | 어디 | 항상표시 별도패널은 과부하 → 타일맵 그룹 + 접이식으로. |

**왜 조합인가(단독 기각):** (a)단독=거시적이나 개별식별 불가 → 타일맵 보강. (b)단독=스냅샷뿐, 거시추세는
애니메이션 없인 안 보임 → 히어로 보강. (c)단독=타이밍뿐, 재고·식별 없음 → 보조역. (d)를 히어로화=너무 추상.
셋의 합 = 얼마나+누가+언제 = "구분+천천히 관찰"을 정확히 커버.

### 3.2 정보 위계 (코호트/집계 모드 결과영역, 위→아래)

1. **연도 컨트롤 바:** 슬라이더 2026–2045 + ▶재생/⏸/속도 + **연동 헤드라인** "이 조건이면 **{year}년 폐교위험 N개교**(2026 대비 +K) · 심각 X·위험 Y · 범위 [lo~hi]" — 슬라이더/재생에 실시간 갱신.
2. **히어로 스택영역**(전국 344, 밴드순 아래→위: unrated(muted)·stable(good)·caution(warning)·atrisk(serious)·**critical(critical, 최상단 적색 캡**) + 정직성 3구간 배경 + 세로 **플레이헤드** + atRiskRange 확대밴드 + 하단 (c) incidence 막대.
3. **권역 타일맵**(슬라이더연도 스냅샷, 수도권/광역권/지방권 3그룹, 그룹내 등급강도→소진연도 정렬로 "위험 프런티어" 경계가 전진; 직전 슬라이더연도 대비 등급변화 타일 펄스 하이라이트).
4. **기존 우패널**(학교별 배지 테이블·근거게이트)을 **슬라이더 연도로 필터**(그 해 grade·소진예상).
5. **정직성 푸터**(기존 `closureHonestyCard` 재사용) + "정착 10년 기준 N개교"를 참조수치로 병기.

밴드 정렬을 unrated 바닥·critical 최상단으로 둔 이유: 상단 적색 캡이 좌→우로 **부풀어 오르는** 실루엣이
"위험 대학 증가"의 가장 직관적 알람. 총합 344 고정이라 100% 구성차트로 읽힌다.

### 3.3 성능 — 사전계산/판독 분리 (슬라이더 실시간 요건)

핵심: **파라미터 변경 시 1회만 무거운 재계산**, 슬라이더·재생은 **사전계산 판독뿐**.

- **파라미터 변경(디바운스):** `SIM.closureAggregateTimeline` 1회 → `[344 × 21]` 등급행렬 + 연도별 집계 저장.
  비용 추정: d14 실측 `344×2×(10년 파이프라인)=14~22ms` → (school×scenario×10y-pipe)≈32µs.
  20년은 ≈2×(64µs). 타임라인=`344 × (현재 + 기준선) × 64µs` ≈ **44ms** + grade(t) 21×344 매트릭스평가(≈0.5µs)≈무시.
  디바운스(110ms, `app.js:2523`) 뒤 44ms → 사실상 즉시.
- **슬라이더 이동/재생 프레임:** 사전계산 `bySchool[i].grades[year]` 판독 → 타일 재색칠 O(344) + 헤드라인 O(1).
  재계산 전무 → 60fps 여유.
- **자료구조:**
  ```
  timeline = {
    years:[2025..2045],
    perYear:[{year, counts{stable..unrated}, atRiskOrWorse, atRiskRange:[lo,hi],
              byRegion{}, byScale{}, newAtRisk, downgradedVsBase}],
    bySchool:[{idx, grades:[g_2025..g_2045], depletionYear, firstAtRiskYear}],
    honesty:{base:2025, validatedTo:2029, driftDataTo:2040, horizonTo:2045}
  }
  ```
- **추가 최적화(필요시):** 기준선 재계산 생략(전이를 r=0 대신 **2026 기준연도** 대비로 정의) / project 결과를
  recalcSeries·grade(t)가 공유(재투영 금지) / 44ms가 튀면 Web Worker(불필요 예상). 슬라이더는 어떤 경우도 재계산 안 함.
- **재생 버튼:** `raf`(기존 `app.js:1764` 폴백 패턴 재사용), 2~4년/초(20년 스윕 ≈5~10초 = "천천히"),
  **`prefers-reduced-motion` 존중**(슬라이더가 접근가능 폴백, 재생은 향상). 재생 중 슬라이더 조작 시 일시정지.

## 4. 파라미터 연동 + 기존 debounce 결합

- 시나리오 슬라이더(r·β·γ·η·dropout·piRate·λ값)의 `onChange`는 이미 `scheduleUpdate()`(110ms 디바운스, `app.js:2541-2544,2523`).
  선택형(profile·fillMode·piMode·λ/corp 토글)은 `render()` 전체 재렌더.
- **결합:** `updateResults()`(cohort분기)에서 `buildSimAggregate`가 **정적 집계 대신 타임라인 파이프라인**을 호출하도록 교체:
  `timeline = SIM.closureAggregateTimeline(SIMD, timelineParams(P), opts)` → 상태 `S.sim_tl = timeline` 저장 후 뷰 그리기.
  타임라인 컴퓨트는 디바운스된 이 경로에만 걸리고, **슬라이더/재생은 `S.sim_tl` 판독만** 하므로 파라미터 재계산과 완전 분리.
- `timelineParams(P)` = `{…P, t0:2025, horizon:20, rStart:P.t0}`(§2.4). 슬라이더 연도 상태 `S.sim_tl_year`(기본 2026 또는
  마지막 조작값), 재생 상태 `S.sim_tl_play`. 파라미터 변경으로 타임라인 재계산돼도 `S.sim_tl_year`는 보존(관찰 위치 유지).
- 코호트 집계대상(전국/수도권/경쟁, `SIM_AGG_POP`)은 타임라인의 학교 부분집합 필터로 승계.

## 7. 리스크 · 검토한 대안과 버린 이유

**리스크·가정**
1. **20년 외삽의 근본한계:** 4년 검증을 20년으로 늘리는 것 자체가 관찰용. σ(t) 확대·3구간 라벨·"관찰용 외삽"으로 정직 처리하되,
   **통계적 예측이 아님**을 UI가 반복 고지. (미확인: 20년 실측 검증은 불가능 — 구조적으로 추정.)
2. **인구 2041+ 동결의 방향성 편향:** 동결은 위험을 **과소** 표현(한국 18세 인구는 2041+에도 실제 감소 추세일 가능성).
   근거 없는 감소 주입 회피를 우선해 동결+라벨 채택, `driftTail:'lastRate'` 감쇠는 민감도 토글로만. (추정: 실제 추세는 계속 하락.)
3. **완충축 이중정의:** closureGrade(t0 총여력) vs grade(t)(잔여여력)는 **의도적** 상이. 헤드라인은 grade(t) 기반 N(t),
   정착 참조수치는 병기. 사용자 혼동 방지 카피 필요.
4. **bt 커버리지 불균질:** 백테스트교만 mape_rev 보유 → 개별 확대밴드는 그들만, 나머지는 구간음영만. 집계밴드도 bt 보유교 섭동 기반(하한 성격).
5. **rStart 디커플의 π/드리프트 원점:** 온셋만 분리하고 원점은 2025 고정 — 온셋≠2025일 때 "감축 전 등록금인상·인구감소는 계속"이라는
   해석을 명시(모델 정합엔 문제 없음).

**검토한 대안(버림)**
- **grade(t)를 정착판정 재사용(매년 horizon만큼 재투영):** 344×20×project = O(20²) 폭증, 앞구간 불일치 위험. → project 1회 + 연도 스냅샷 판독으로 대체.
- **슬라이더마다 재계산:** 실시간 불가(수십 ms×프레임). → 사전계산 [344×21] 판독 분리.
- **히스테리시스 기본 적용:** 결정론·단조라 근거 없음, 진짜 전이(U자 회복) 은폐 위험. → 기본 OFF, D7 증거 시 확정지연 가드.
- **charts.js에 stackArea 신설:** 소유권 밖·충돌 위험. → 인라인 SVG 자체구현.
- **t0를 그대로 이동(기저=온셋 유지):** 2025~2029 소실·달력 라벨 붕괴. → rStart 가법 디커플.
- **인구 2041+ 감소 외삽 기본값:** 데이터 없는 위험 과장. → 동결+라벨 기본, 감쇠는 토글.
- **지역 소계 라인 상시패널(d 단독 승격):** 과부하·추상. → 타일맵 권역 그룹핑으로 흡수.
